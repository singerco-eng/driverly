# DR-004: Credential Submission

> **Last Updated:** 2026-01-16  
> **Status:** Draft  
> **Phase:** 3 - Driver Core

---

## Overview

Credential Submission is the driver's self-service interface for submitting and managing their credentials. Drivers upload documents, capture photos, sign agreements, and fill out forms to meet the requirements defined in AD-005 (Credential Types). This feature works alongside AD-006 (Credential Review) where admins approve/reject submissions.

**Key Principle:** Drivers never modify existing submissions - they always create new submissions. Historical records are preserved for audit purposes.

---

## Credential Organization

### Two Main Sections

1. **Driver Credentials** - Attached to the driver (license, background check, certifications)
2. **Vehicle Credentials** - Attached to vehicles (insurance, registration, inspections)

### Within Each Section

- **Global Credentials** - Required for all work
- **Broker-Specific Credentials** - Required for specific brokers (grouped by broker)

### Visibility Rules

| Credential Type | Visible When |
|-----------------|--------------|
| Global Driver | Always |
| Global Vehicle | Driver has at least one vehicle |
| Broker Driver | Assigned to OR requested broker |
| Broker Vehicle | Assigned/requested broker + has vehicle |

## Data Model

### Credential Submission

From AD-005, the `driver_credentials` and `vehicle_credentials` tables store submissions:

```sql
-- Submission data stored in credential instance
-- document_url, signature_data, form_data, entered_date, notes

-- Add history tracking
CREATE TABLE credential_submission_history (
  id                  uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  credential_id       uuid NOT NULL,  -- References driver_credentials or vehicle_credentials
  credential_table    text NOT NULL CHECK (credential_table IN ('driver_credentials', 'vehicle_credentials')),
  
  -- Snapshot of submission at time of review
  submission_data     jsonb NOT NULL,  -- Copy of document_url, form_data, etc.
  status              text NOT NULL,
  
  -- Review info
  reviewed_at         timestamptz,
  reviewed_by         uuid REFERENCES users(id),
  review_notes        text,
  rejection_reason    text,
  
  created_at          timestamptz NOT NULL DEFAULT now()
);

CREATE INDEX idx_credential_history_credential ON credential_submission_history(credential_id, credential_table);

-- RLS
ALTER TABLE credential_submission_history ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Drivers can view own credential history"
  ON credential_submission_history FOR SELECT
  USING (
    credential_id IN (
      SELECT id FROM driver_credentials 
      WHERE driver_id = (SELECT id FROM drivers WHERE user_id = auth.uid())
    )
    OR credential_id IN (
      SELECT vc.id FROM vehicle_credentials vc
      JOIN vehicles v ON vc.vehicle_id = v.id
      WHERE v.owner_driver_id = (SELECT id FROM drivers WHERE user_id = auth.uid())
    )
  );
```

## User Stories

### Driver Stories

1. **As a Driver**, I want to see all my required credentials organized clearly.

2. **As a Driver**, I want to see my progress toward completing all credentials.

3. **As a Driver**, I want to upload documents for credentials that require them.

4. **As a Driver**, I want to capture photos using my phone camera.

5. **As a Driver**, I want to sign agreements electronically.

6. **As a Driver**, I want to fill out forms for credentials that require them.

7. **As a Driver**, I want to add notes with my submissions.

8. **As a Driver**, I want to see what I previously submitted.

9. **As a Driver**, I want to know why a credential was rejected so I can fix it.

10. **As a Driver**, I want to be warned when credentials are expiring.

11. **As a Driver**, I want to re-submit credentials before they expire.

12. **As a Driver**, I want to see my vehicle credentials grouped by vehicle.

13. **As a Driver**, I want to see my submission history.

## Business Rules

1. **No modification:** Drivers always create new submissions, never modify existing

2. **History preserved:** All submissions kept for audit trail

3. **Visibility:** Only show credentials for assigned/requested brokers

4. **Expiration threshold:** Warning days set per credential type by admin

5. **Proactive renewal:** Can submit new credential before current expires

6. **Admin-verified:** Driver cannot submit - just waits for admin to mark complete

7. **Multiple files:** Document upload and photo can have multiple files

8. **Draft saving:** Form submissions can be saved as draft

9. **Signature binding:** E-signatures include timestamp and are legally binding

10. **File limits:** 50MB per file, common formats only
