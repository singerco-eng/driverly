# AD-001: Driver Applications

> **Last Updated:** 2026-01-16  
> **Status:** Draft  
> **Phase:** 2 - Admin Core

---

## Overview

Driver Applications enables prospective drivers to apply to work for a tenant company. The application is a multi-step wizard with auto-save, accessible via a public URL. Admins review applications and approve or reject them.

**Key Concepts:**
- **Application Approved** = Driver can access portal, begin credentialing
- **Driver Active** = All required credentials approved, can accept trips (per broker)

---

## User Stories

### Driver Stories

1. **As a prospective driver**, I want to find and access the application for a company so that I can apply to work for them.

2. **As a prospective driver**, I want to save my progress and resume later so that I don't lose my work.

3. **As a prospective driver**, I want to create an account during the application so that I can log in and check my status.

4. **As an applicant**, I want to see my application status so that I know where I am in the process.

5. **As an applicant**, I want to withdraw my application if I change my mind.

6. **As a rejected applicant**, I want to re-apply after the waiting period so that I can try again.

### Admin Stories

7. **As an Admin**, I want to see all pending applications so that I can review them.

8. **As an Admin**, I want to review an application's details so that I can make an informed decision.

9. **As an Admin**, I want to approve an application so that the driver can begin onboarding.

10. **As an Admin**, I want to reject an application with a reason so that the applicant understands why.

11. **As an Admin**, I want to add internal notes to an application for my records.

## Data Model

### Application Status on `drivers` Table

The `drivers` table (defined in `02-DATABASE-SCHEMA.md`) handles application state:

```sql
-- drivers.application_status values
'draft'        -- Started, not submitted (has auto-saved progress)
'pending'      -- Submitted, awaiting review
'under_review' -- Admin is actively reviewing
'approved'     -- Accepted, can access portal
'rejected'     -- Declined
'withdrawn'    -- Driver cancelled application
```

### Additional Fields for Applications

```sql
-- Add to drivers table
ALTER TABLE drivers ADD COLUMN IF NOT EXISTS
  -- Application tracking
  application_submitted_at timestamptz,
  
  -- License photos
  license_front_url text,
  license_back_url text,
  
  -- Experience/notes from applicant
  experience_notes text,
  referral_source text,
  
  -- Rejection tracking
  rejected_at timestamptz,
  can_reapply_at timestamptz,  -- 14 days after rejection
  
  -- EULA acceptance
  eula_accepted_at timestamptz,
  eula_version text;

-- Application drafts (for auto-save before submission)
CREATE TABLE application_drafts (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  company_id      uuid NOT NULL REFERENCES companies(id) ON DELETE CASCADE,
  
  -- Draft data (JSON blob of form state)
  form_data       jsonb NOT NULL DEFAULT '{}',
  current_step    integer NOT NULL DEFAULT 1,
  
  -- Tracking
  created_at      timestamptz NOT NULL DEFAULT now(),
  updated_at      timestamptz NOT NULL DEFAULT now(),
  
  UNIQUE(user_id, company_id)
);

-- Indexes
CREATE INDEX idx_application_drafts_user ON application_drafts(user_id);
CREATE INDEX idx_application_drafts_company ON application_drafts(company_id);

-- RLS
ALTER TABLE application_drafts ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can manage own drafts"
  ON application_drafts FOR ALL
  USING (user_id = auth.uid());
```

## Acceptance Criteria

### AC-1: Application Access

- [ ] Application available at `/apply/[company-slug]`
- [ ] Invalid slug shows error page
- [ ] Inactive company shows error page
- [ ] Active company shows application wizard

### AC-2: Account Creation (Step 1)

- [ ] Can enter email and password
- [ ] Password validation (8+ chars, upper, lower, number)
- [ ] On continue, auth user created
- [ ] User signed in automatically
- [ ] Draft record created
- [ ] If email exists for this company â†’ prompt to log in

### AC-3: Personal Information (Step 2)

- [ ] All required fields enforced
- [ ] Date of birth validates 18+ years old
- [ ] State is dropdown of US states
- [ ] Data auto-saves on blur

### AC-4: Employment Type (Step 3)

- [ ] Must select W2 or 1099
- [ ] Selection stored
- [ ] Determines if Step 5 shown

### AC-5: License Information (Step 4)

- [ ] All fields required
- [ ] Expiration must not be past
- [ ] Can take photo with camera
- [ ] Can upload photo from file
- [ ] Photos uploaded to Supabase Storage
- [ ] Both front and back required

### AC-6: Vehicle Information (Step 5 - 1099 only)

- [ ] Only shown for 1099 applicants
- [ ] Can skip ("I'll add later")
- [ ] If not skipped, required fields enforced
- [ ] Vehicle data stored

### AC-7: Review & Submit (Step 6)

- [ ] Shows all entered information
- [ ] Edit buttons return to correct step
- [ ] Optional fields available (experience, referral)
- [ ] Must accept EULA to submit
- [ ] On submit:
  - [ ] Driver record created with `application_status: 'pending'`
  - [ ] Vehicle record created (if provided)
  - [ ] Draft deleted
  - [ ] `application_submitted_at` set
  - [ ] Confirmation email sent
  - [ ] Redirects to status page

### AC-8: Auto-Save

- [ ] Form data saves to draft on field blur
- [ ] Form data saves every 30 seconds
- [ ] Shows save indicator
- [ ] On return, form restored from draft
- [ ] Current step restored

### AC-9: Application Status

- [ ] Logged-in applicants see their status
- [ ] Status updates reflected immediately
- [ ] Approved applicants see link to portal
- [ ] Rejected applicants see reason and re-apply date

### AC-10: Withdraw Application

- [ ] Pending applicants can withdraw
- [ ] Confirmation required
- [ ] Status changes to `withdrawn`
- [ ] Cannot undo withdrawal

### AC-11: Admin Applications List

- [ ] Shows all applications for company
- [ ] Can filter by status
- [ ] Can filter by employment type
- [ ] Can search by name/email
- [ ] Pending applications shown first

### AC-12: Admin Review

- [ ] Shows all application details
- [ ] Shows license photos (viewable/zoomable)
- [ ] Can add internal notes
- [ ] Can approve application
- [ ] Can reject with reason

### AC-13: Approval Flow

- [ ] On approve:
  - [ ] `application_status` â†’ `approved`
  - [ ] `approved_at` and `approved_by` set
  - [ ] Driver can access portal
  - [ ] Approval email sent

### AC-14: Rejection Flow

- [ ] Must select rejection reason
- [ ] Optional additional details
- [ ] On reject:
  - [ ] `application_status` â†’ `rejected`
  - [ ] `rejection_reason` stored
  - [ ] `rejected_at` set
  - [ ] `can_reapply_at` set (14 days out)
  - [ ] Rejection email sent with reason

### AC-15: Re-Application

- [ ] Rejected applicants can re-apply after 14 days
- [ ] Before 14 days â†’ shows wait time
- [ ] After 14 days â†’ can start new application
- [ ] Old application data not pre-filled

## API/Functions

### Edge Function: Submit Application

```typescript
// supabase/functions/submit-application/index.ts

interface ApplicationSubmission {
  companyId: string;
  personalInfo: {
    fullName: string;
    phone: string;
    dateOfBirth: string;
    address: Address;
  };
  employmentType: 'w2' | '1099';
  license: {
    number: string;
    state: string;
    expiration: string;
    frontUrl: string;
    backUrl: string;
  };
  vehicle?: {
    type: string;
    make: string;
    model: string;
    year: number;
    licensePlate: string;
    color?: string;
  };
  experienceNotes?: string;
  referralSource?: string;
  eulaVersion: string;
}

Deno.serve(async (req) => {
  const supabaseAdmin = createClient(/*...*/);
  
  // Get authenticated user
  const { data: { user } } = await supabaseAdmin.auth.getUser(/*...*/);
  if (!user) return unauthorized();
  
  const submission: ApplicationSubmission = await req.json();
  
  // Validate age
  const age = calculateAge(submission.personalInfo.dateOfBirth);
  if (age < 18) {
    return error('Must be 18 or older to apply');
  }
  
  // Validate license not expired
  if (new Date(submission.license.expiration) < new Date()) {
    return error('Driver license is expired');
  }
  
  // Create driver record
  const { data: driver } = await supabaseAdmin
    .from('drivers')
    .insert({
      user_id: user.id,
      company_id: submission.companyId,
      employment_type: submission.employmentType,
      date_of_birth: submission.personalInfo.dateOfBirth,
      license_number: submission.license.number,
      license_state: submission.license.state,
      license_expiration: submission.license.expiration,
      license_front_url: submission.license.frontUrl,
      license_back_url: submission.license.backUrl,
      experience_notes: submission.experienceNotes,
      referral_source: submission.referralSource,
      application_status: 'pending',
      application_submitted_at: new Date().toISOString(),
      eula_accepted_at: new Date().toISOString(),
      eula_version: submission.eulaVersion,
    })
    .select()
    .single();
  
  // Update user record
  await supabaseAdmin
    .from('users')
    .update({
      full_name: submission.personalInfo.fullName,
      phone: submission.personalInfo.phone,
      // address fields...
    })
    .eq('id', user.id);
  
  // Update auth metadata with driver_id
  await supabaseAdmin.auth.admin.updateUserById(user.id, {
    app_metadata: {
      ...user.app_metadata,
      driver_id: driver.id,
    },
  });
  
  // Create vehicle if provided
  if (submission.vehicle) {
    const { data: vehicle } = await supabaseAdmin
      .from('vehicles')
      .insert({
        company_id: submission.companyId,
        owner_type: 'driver',
        owner_driver_id: driver.id,
        vehicle_type: submission.vehicle.type,
        make: submission.vehicle.make,
        model: submission.vehicle.model,
        year: submission.vehicle.year,
        license_plate: submission.vehicle.licensePlate,
        color: submission.vehicle.color,
        status: 'active',
      })
      .select()
      .single();
    
    // Create assignment
    await supabaseAdmin
      .from('driver_vehicle_assignments')
      .insert({
        driver_id: driver.id,
        vehicle_id: vehicle.id,
        company_id: submission.companyId,
        assignment_type: 'owned',
        is_primary: true,
        assigned_by: user.id,
      });
  }
  
  // Delete draft
  await supabaseAdmin
    .from('application_drafts')
    .delete()
    .eq('user_id', user.id)
    .eq('company_id', submission.companyId);
  
  // Send confirmation email
  await sendApplicationConfirmationEmail(user.email, submission.companyId);
  
  return Response.json({ success: true, driverId: driver.id });
});
```

## Business Rules

1. **Age requirement:** Must be 18+ years old
2. **License validity:** License cannot be expired at time of application
3. **One application per company:** Cannot have multiple pending applications to same company
4. **Multi-company:** Can apply to multiple companies
5. **Re-application:** 14-day waiting period after rejection
6. **Draft expiration:** Drafts never expire (can resume anytime)
7. **Employment type:** Selected at application, determines credential requirements later
