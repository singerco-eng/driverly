# AD-006: Credential Review

> **Last Updated:** 2026-01-16  
> **Status:** Draft  
> **Phase:** 4 - Admin Operational

---

## Overview

Credential Review allows Admins and Coordinators to review, approve, and reject credential submissions from drivers. This feature also handles admin-verified credentials where the admin marks completion directly. The review can be done from a central credentials page or from individual driver/vehicle detail pages.

**Key Principle:** All credential submissions go through admin review before becoming valid. Admins have full visibility into submission history and can add notes for internal tracking.

## User Stories

### Admin/Coordinator Stories

1. **As an Admin**, I want to see all pending credential submissions so I can review them.

2. **As an Admin**, I want to approve a credential submission so the driver becomes eligible.

3. **As an Admin**, I want to reject a credential with a reason so the driver knows what to fix.

4. **As an Admin**, I want to add private notes that only staff can see.

5. **As an Admin**, I want to view the submitted documents/photos in a modal.

6. **As an Admin**, I want to set or adjust the expiration date when approving.

7. **As an Admin**, I want to mark admin-verified credentials as complete.

8. **As an Admin**, I want to see credential history to understand past submissions.

9. **As an Admin**, I want to filter credentials by status, type, driver, and broker.

10. **As an Admin**, I want to see which credentials are expiring soon.

11. **As an Admin**, I want to review vehicle credentials the same way as driver credentials.

## Business Rules

1. **Review required:** All driver-submitted credentials need approval before becoming valid

2. **Rejection reason required:** Must provide reason that driver will see

3. **Two types of notes:** Review notes (public to driver) and internal notes (staff only)

4. **Expiration handling:** 
   - Prefilled from credential type configuration
   - Can be overridden by reviewer
   - "Never expires" for one-time credentials

5. **Admin-verified:** No driver submission - admin marks complete directly

6. **Unverify option:** Admin-verified credentials can be unverified if needed

7. **History preserved:** All review actions logged for audit

8. **Coordinator same as admin:** For credential review, coordinators have same permissions

const { data: pendingCredentials } = await supabase

### Approve Credential

```typescript
async function approveCredential(
  credentialId: string,
  data: {
    expiresAt?: string;
    reviewNotes?: string;
    internalNotes?: string;
  }
) {
  const { data: credential } = await supabase
    .from('driver_credentials')
    .update({
      status: 'approved',
      expires_at: data.expiresAt,
      reviewed_at: new Date().toISOString(),
      reviewed_by: currentUser.id,
      review_notes: data.reviewNotes,
    })
    .eq('id', credentialId)
    .select()
    .single();
  
  // Log to history
  await supabase
    .from('credential_submission_history')
    .insert({
      credential_id: credentialId,
      credential_table: 'driver_credentials',
      submission_data: {}, // Snapshot of current submission
      status: 'approved',
      reviewed_at: new Date().toISOString(),
      reviewed_by: currentUser.id,
      review_notes: data.internalNotes, // Internal notes go to history
    });
  
  return credential;
}
```
